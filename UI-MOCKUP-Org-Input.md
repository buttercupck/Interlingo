## Supabase Schema Needed

> **Architecture Note:** This table stores OPERATIONAL DATA ONLY (addresses, credentials, contacts).
> Workflow instructions (procedures, special requirements, calendar minimums) live in markdown config files.

```sql
-- organizations table structure
CREATE TABLE organizations (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),

  -- Basic Info
  name TEXT NOT NULL,
  abbreviation TEXT,
  location_type TEXT CHECK (location_type IN ('court', 'law_office', 'agency', 'other')),
  status TEXT CHECK (status IN ('active', 'draft', 'inactive')),

  -- Location (operational data for address/map display)
  street_address TEXT,
  city TEXT,
  state TEXT,
  zip TEXT,

  -- Modalities (boolean flags for what this org offers)
  offers_zoom BOOLEAN DEFAULT false,
  offers_in_person BOOLEAN DEFAULT false,
  offers_phone BOOLEAN DEFAULT false,

  -- Zoom Credentials (operational data for joining meetings)
  zoom_link TEXT,
  zoom_meeting_id TEXT,
  zoom_passcode TEXT,
  zoom_waiting_room BOOLEAN,

  -- Phone Operational Data
  phone_number TEXT,
  conference_id TEXT,

  -- Contacts (operational data for who to call)
  primary_contact_name TEXT,
  primary_contact_phone TEXT,
  primary_contact_email TEXT,
  billing_contact_name TEXT,
  billing_contact_phone TEXT,
  billing_contact_email TEXT,

  -- Internal Notes (not sent to interpreters)
  billing_notes TEXT,

  -- Metadata
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- NOTE: The following are NOT in this table (managed in markdown configs):
-- - Check-in/check-out procedures (workflow instructions)
-- - Dress code requirements (workflow instructions)
-- - Calendar minimums (workflow rules)
-- - 3-hour minimum requirements (workflow rules)
-- - Mileage reimbursement procedures (workflow instructions)
-- - Standing appointment patterns (workflow patterns)
-- - Security procedures (workflow instructions)
-- - Parking instructions (workflow instructions)
```
